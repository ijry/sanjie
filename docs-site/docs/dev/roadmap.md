# 开发路线

完整实施计划位于：

```text
docs/superpowers/plans/2026-06-18-sanjie-mobile-full-implementation.md
```

## 阶段 1：开源项目骨架

- 根目录 README、License、协作规则、配置样例。
- VitePress 文档站。
- GitHub Actions CI 和 GitHub Pages 文档发布。
- 贡献指南、安全说明、变更日志、Issue/PR 模板。

## 阶段 2：后端基础

- Go 模块初始化。
- SQLite 连接、迁移、种子数据。
- 统一响应格式和演示身份。
- 健康检查、用户、工作台、告警、审计日志。
- 当前版本已实现完整 Go + SQLite API，并通过 `go test ./...` 验证。

## 阶段 3：核心业务 API

- 勾魂任务。
- 投胎队列。
- 审批中心。
- 生死簿。
- 十八层地狱。
- 愿望工单。
- 孟婆汤。

## 阶段 4：移动端基础

- uni-app 工程。
- uview-plus 配置。
- API 封装。
- 用户身份切换。
- 工作台和底部 tab。

## 阶段 5：移动端业务页面

- 勾魂任务列表和详情。
- 投胎队列列表和详情。
- 审批中心列表和详情。
- 生死簿查询和详情，支持冻结、改寿申请和审计记录展示。
- 十八层地狱看板和刑期详情，支持楼层分流和刑期复核申请。
- 愿望工单详情支持转派、办结、驳回和结果备注。
- 孟婆汤、审计日志和工作台快捷入口。
- 当前版本已实现 uni-app + uview-plus H5 页面，并通过 `npm run build:h5` 验证。

## 阶段 6：联调验收

- 后端测试通过。
- H5 构建通过。
- 文档构建通过。
- 根目录 `npm run verify` 可一次性执行后端测试/构建、H5 构建和文档构建。
- 手工验收完整业务链路。
